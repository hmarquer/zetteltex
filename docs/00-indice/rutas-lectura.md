# Rutas de lectura

Este documento propone recorridos concretos segun tu perfil.

## Ruta A: Usuario no tecnico (usar la herramienta)

Objetivo:
Aprender el flujo diario para crear, mantener y exportar notas/proyectos.

Orden recomendado:
1. [Inicio rapido detallado](../01-guia-usuario/inicio-rapido.md)
2. [Flujo diario](../01-guia-usuario/flujo-diario.md)
3. [Solucion de problemas](../01-guia-usuario/solucion-problemas.md)
4. [Preguntas frecuentes](../01-guia-usuario/preguntas-frecuentes.md)
5. [Referencia de comandos](../03-comandos/00-referencia.md)
6. [Catalogo por comando](../03-comandos/README.md)

Resultados esperados:
- crear una nota,
- sincronizar,
- renderizar PDF,
- exportar a Markdown,
- usar la interfaz fuzzy.

## Ruta B: Usuario tecnico (entender como funciona)

Objetivo:
Entender la organizacion interna del sistema para mantener y extender la CLI.

Orden recomendado:
1. [Guia tecnica (indice)](../02-guia-tecnica/README.md)
2. [Modelo de workspace](../02-guia-tecnica/modelo-workspace.md)
3. [Sincronizacion](../02-guia-tecnica/sincronizacion.md)
4. [Modelo de datos](../02-guia-tecnica/modelo-datos.md)
5. [Pipeline de render](../02-guia-tecnica/pipeline-render.md)
6. [Pipeline de export](../02-guia-tecnica/pipeline-export.md)
7. [Arquitectura](../04-arquitectura/README.md)
8. [API para desarrollo](../02-guia-tecnica/api.md)
9. [Catalogo por comando](../03-comandos/README.md)
10. [Funciones de codigo (indice)](../05-funciones-codigo/README.md)

Resultados esperados:
- ubicar responsabilidades por crate,
- entender el flujo de ejecucion por comando,
- localizar modulos para cambios de codigo.
